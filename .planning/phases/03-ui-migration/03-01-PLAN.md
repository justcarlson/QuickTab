  <files>entrypoints/popup/style.css</files>
  <action>
Replace the placeholder CSS with a complete design system using CSS custom properties.

Include:
1. :root variables for light mode (default):
   - --bg: #f7f7f2 (background)
   - --surface: #ffffff (card/surface)
   - --text: #1b1d1a (primary text)
   - --muted: #5a5f56 (secondary text)
   - --border: #e5e6dd (borders)
   - --accent: #c8f05a (accent/active state)
   - --accent-dark: #9fd138 (accent hover)
   - --accent-soft: #eef7d6 (accent background)

2. @media (prefers-color-scheme: dark) override:
   - --bg: #111311
   - --surface: #1c1f1c
   - --text: #f2f4ef
   - --muted: #a4aba0
   - --border: #2e332d
   - --accent: #c8f05a (same)
   - --accent-dark: #9fd138 (same)
   - --accent-soft: #20300f

3. Base styles:
   - box-sizing: border-box on *
   - body: font-family system-ui stack, 13px, var(--text), var(--bg)
   - min-width: 260px for popup container

4. Loading state:
   - .loading class with opacity: 0.5 and pointer-events: none

5. Typography:
   - h1: 16px, font-weight 600
   - h2: 10px uppercase, letter-spacing 0.1em, var(--muted), border-bottom

6. Mode option styles (radio buttons as cards):
   - .mode-option: flex, justify-between, padding 10px 12px, border-radius 8px
   - .mode-option:hover: background var(--accent-soft)
   - .mode-option input: visually hidden (position absolute, opacity 0)
   - .mode-option .indicator: 20px circle, border 2px solid var(--border)
   - .mode-option input:checked + .indicator: border and background var(--accent)
   - .mode-option input:focus-visible + .indicator: outline 2px solid var(--accent)

7. Help section link styles:
   - ul: list-style none, padding/margin 0
   - li: padding 8px 12px, border-radius 8px
   - li:hover: background var(--accent-soft)
   - a: color var(--text), text-decoration none
   - a:hover: color var(--accent-dark)

This CSS will be imported by both popup and welcome page stylesheets.
